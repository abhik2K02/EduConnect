# Lecture Assistant - Frontend Architecture Guide

## Overview
The frontend of the Lecture Assistant application is built using **React**, powered by **Vite** for fast bundling, and styled seamlessly with **Tailwind CSS**. It serves as the primary interface for both Teachers (to record audio, capture live transcripts, and query knowledge) and Students (to review lectures, summaries, and chat with AI about the content).

## 1. Routing & Layout (`App.jsx`)
The application uses **React Router DOM** for navigation. The routing maps specific roles to their respective dashboards:
- `/login` & `/register`: Public authentication routes.
- `/teacher`: Protected route rendering the `TeacherDashboard`.
- `/student`: Protected route rendering the `StudentArchive`.
- `/student/lecture/:id`: Dynamic route for viewing a specific lecture's details via `LectureClassView`.
- `/admin`: Reserved dashboard for administrative tasks.

A `<ProtectedRoute>` component wraps these role-based routes to ensure that only authenticated users with the correct `userRole` (from Firebase/Context) can access them. Additionally, there is an `<ErrorBoundary>` that catches UI errors to prevent white-screen crashes.

## 2. State Management & Authentication (`context/AuthContext.js`)
Global authentication state is managed using the React Context API (`AuthProvider`). It interfaces directly with **Firebase Auth** to track the current user and their specific role (Teacher vs. Student).
- Provides `currentUser`, `userRole`, and methods like `login`, `register`, and `logout` to any consuming component via the `useAuth()` custom hook.

## 3. Key Components

### `TeacherDashboard.jsx`
The central hub for educators. Its main responsibilities include:
1. **Audio Recording**: Utilizes the browser's `MediaRecorder` API to capture microphone input as a WebM audio blob.
2. **Live Transcription (Deepgram)**: While recording, audio chunks are sent to Deepgram over a WebSocket connection to generate real-time subtitles/transcripts (`is_final` and `interim_results`).
3. **Cloud Uploading**: Upon stopping the recording, the finalized audio file is uploaded to the backend/Firebase via a `multipart/form-data` POST request.
4. **Live Edu Assistant**: Offers a "Hey Edu" manual voice trigger utilizing the native HTML5 `SpeechRecognition` API. Teachers can ask questions mid-lecture, which are routed to the backend for AI processing and return relevant YouTube search queries or contextual notes.

### `StudentArchive.jsx`
The primary dashboard for students. It queries the backend/Firebase to fetch a list of available lectures that the student has access to. It displays these as interactive cards or list items, serving as an entry point to individual lecture reviews.

### `LectureClassView.jsx`
The interactive learning hub. When a student selects a lecture, this view opens to present:
- The **Media Player** (to listen to the uploaded audio).
- The **Generated Summary** (provided by Gemini 1.5 Flash on the backend).
- The **Full Transcript**.
- An **AI Chat Interface** where students can query the specific content of that lecture using Pinecone vector embeddings for RAG (Retrieval-Augmented Generation).

### `TeacherChatbot.jsx`
A dedicated chat interface nested within the Teacher Dashboard, allowing educators to converse with their AI assistant regarding lecture planning, past materials, or general queries.

## 4. API & External Services
- **Firebase**: Used for Authentication (AuthContext) and potentially Storage (importing `storage` from `firebase.js`).
- **Deepgram WebSockets**: Directly connected from the frontend in the Teacher Dashboard for ultra-low latency transcription.
- **Backend Communication**: Components make standard `fetch` API calls to `http://localhost:5000/api/...` to trigger heavy tasks like vector processing, AI summaries, or data retrieval.

## 5. Styling
The project heavily leverages **Tailwind CSS**. You'll see utility classes used for responsive design, flexbox/grid layouts, shadows, and spacing applied directly within the JSX `className` attributes.
